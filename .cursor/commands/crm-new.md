ИНТЕРФЕЙС CRM:

1. Создать админ-панель CRM с разделами:
   - Готовое жилье (продажа)
   - Готовое жилье (аренда)
   - Новостройки

2. Для каждого раздела создать:
   - Список объектов (таблица с фильтрами и поиском)
   - Форму добавления объекта
   - Форму редактирования объекта
   - Кнопку удаления объекта

3. Форма добавления/редактирования должна содержать:
   - Все поля из таблицы БД в виде соответствующих input-элементов
   - Для ENUM-полей использовать select или radio buttons
   - Для BOOLEAN-полей использовать checkbox
   - Для дат использовать input type="date"
   - Для цен и площадей использовать input type="number" с step="0.01"
   - Загрузку множественных изображений (минимум 5 для готового жилья, 10 для новостроек)
   - Возможность отметить главное фото
   - Сортировку фотографий drag-and-drop (на Vanilla JS)

4. Валидация форм:
   - Обязательные поля: название/адрес, район, цена, площади, количество комнат
   - Проверка на корректность числовых значений
   - Проверка наличия минимум 1 фотографии
   - Вывод ошибок на русском языке

5. Обработка форм:
   - Через простой PHP-скрипт без REST API
   - При сохранении записывать данные в MySQL
   - Загрузку файлов реализовать через $_FILES с сохранением в папку /uploads/properties/
   - После успешного сохранения - редирект на список объектов с сообщением "Объект успешно добавлен"
